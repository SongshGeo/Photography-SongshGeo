## ✅ 第二阶段：Figma 设计规范

### 2.1 画板设置
- [ ] **创建设计画板**
  - 桌面端：1920px × 1080px（或 1440px × 900px）
  - 平板端：768px × 1024px
  - 移动端：375px × 667px（iPhone SE）和 390px × 844px（iPhone 12/13）

- [ ] **设置网格系统**
  - 12 列网格（桌面端）
  - 8 列网格（平板端）
  - 4 列网格（移动端）
  - 设置合适的 Gutter 和 Margin

### 2.2 设计系统建立
- [ ] **颜色系统**

  - 辅助色（Secondary Colors）

  - 强调色（Accent Colors）
  - 背景色（Background Colors）
  - 文字颜色（Text Colors）
  - 注意：需要支持深色模式（Dark Mode）

- [ ] **字体系统**
  - 标题字体（Heading Font）
  - 正文字体（Body Font）
  - 字体大小层级（Type Scale）
    - H1: 主标题
    - H2: 副标题/卡片标题
    - Body: 正文
    - Small: 辅助文字
  - 行高（Line Height）
  - 字重（Font Weight）

- [ ] **间距系统（Spacing）**
  - 使用 4px 或 8px 基础单位

  - 组件内边距（Padding）
  - 组件外边距（Margin）

- [ ] **组件库**

  - 卡片（Card）
  - 导航（Navigation）
  - 图标（Icons）

- [ ] **断点定义**
  - 移动端：< 768px

## 📝 重要提醒

### ⚠️ 必须保持的技术约束

1. **文件夹结构不能改变**
   - `content/_index.md` 文件位置和基本结构
   - `themes/gallery/layouts/_default/home.html` 布局文件位置
   - `themes/gallery/layouts/partials/` 组件文件位置
   - **相册组织方式**：`content/trips/` 目录，按地理位置组织（国家/地区/城市）

2. **Hugo 模板语法不能改变**
   - 设计需要适配现有的 Hugo 模板变量
   - 不能删除或修改模板中的 Hugo 函数调用
   - 例如：`{{ .Title }}`, `{{ .RelPermalink }}` 等
   - **标签系统**：使用 `site.Taxonomies.tags` 访问标签数据

3. **数据来源不能改变**
   - 相册数据来自 `content/trips/` 目录（按地理位置组织）
   - **标签数据来自 tags taxonomies**（不再使用 categories）
   - 图片通过 Hugo Resources 系统管理

4. **功能不能减少**
   - 所有现有功能必须保留
   - 可以增强，但不能删除
   - ⚠️ **重要变更**：Categories Navigation 将被 Tags Navigation 替代

### 💡 设计建议

1. **考虑图片的动态性**
   - 图片尺寸可能不一致
   - 背景色会自动提取自图片
   - 设计时要考虑这些动态因素

2. **考虑内容的动态性**
   - 相册数量可能变化（按地理位置组织）
   - **标签数量可能变化**（主题标签和地理标签会随内容增加）
   - 设计时要考虑内容扩展性
   - 标签导航需要支持动态添加新标签

3. **标签系统设计考虑**
   - **主题标签**：用于内容分类（landscape, architecture, nature 等）
   - **地理标签**：用于地点标记（germany, china, japan 等）
   - 考虑标签的视觉区分（如使用不同颜色或图标）
   - 考虑标签筛选功能（点击标签后筛选相册）

4. **性能考虑**
   - 图片使用 Lazy Loading
   - 动画要轻量，不影响性能
   - 考虑移动端性能
   - 标签筛选功能需要考虑性能（可能需要分页或虚拟滚动）

5. **SEO 和可访问性**
   - 保持语义化 HTML 结构
   - 确保文字内容可读
   - 确保交互元素可访问
   - 标签链接需要清晰的 aria-label
